# AI Team Assistant 详细任务清单

## 1. 文档目标

本清单用于将 [SPEC.md](/Users/zhaowei/Documents/就业/AI项目/AI/SPEC.md) 拆解为可执行的开发任务，覆盖：

- 项目初始化
- 后端开发
- 前端开发
- 联调测试
- Docker 与交付
- 风险修正与工程化补齐

默认目标是先交付可运行 MVP，再逐步补齐增强能力。

## 2. 总体里程碑

### M1：项目骨架可运行
- 建立前后端目录结构
- 完成基础依赖安装文件
- 能本地分别启动前端和后端

### M2：RAG 闭环可用
- 支持文档上传、解析、分块、向量入库
- 支持混合检索
- 支持聊天问答流式输出
- 前端聊天页和文档页可完成主要操作

### M3：记忆能力可用
- 短期记忆支持滑动窗口和摘要压缩
- 长期记忆支持提取、查询、删除
- 会话消息支持持久化

### M4：Code Review 与设置页完成
- Code Review 前后端闭环可用
- 设置页支持模型配置和连通性测试
- API 设置接口补齐

### M5：工程化收尾
- 增加基础测试
- 完善异常处理与日志
- 完善 Docker、README、启动说明
- 完成验收与缺陷修复

## 3. 任务拆解原则

- 先做 P0 主链路，再做增强项
- 先保证后端接口真实可用，再做前端精修
- 所有流式接口优先保证稳定性和错误回收
- 所有新增接口都要定义请求、响应、异常格式
- 所有配置项都要区分前端展示值和后端真实持久化值

验收标准：
- 目录结构与规范基本一致
- 所有 Python 包目录包含 `__init__.py`
- 前端目录可被 Vite 正常识别

### 4.2 后端初始化

任务：

- 编写 `.env.example`

验收标准：
- `uvicorn main:app --reload --port 8000` 可启动
- `GET /api/health` 返回 `{"status":"ok"}`

### 4.3 前端初始化

- 接入 Tailwind CSS

补充修正：

验收标准：

- 左侧导航与四个主页面占位可切换

## 5. 第二阶段：后端 RAG 核心链路

### 5.1 ORM 模型实现

任务：
- 实现 `Document`
- 实现 `LongTermMemory`
- 实现 `Conversation`
- 实现 `Message`
- 确保 `create_tables()` 启动时建表

补充要求：
- `Message` 与 `Conversation` 虽然 P0 不完全使用，但必须建好
- 为后续会话持久化预留字段与索引

验收标准：
- 启动后数据库中可见对应表
- 基础插入与查询无报错

### 5.2 文档解析能力

任务：
- 实现 `parse_document(file)`
- 支持 PDF、DOCX、MD、TXT
- 处理编码异常
- 对空文本、不可解析文本返回失败状态

补充要求：
- 对扫描 PDF 解析为空时给出明确错误信息
- 对超大文件设置大小保护或超时保护

验收标准：
- 上传不同文档格式可返回纯文本
- 异常文件进入 `error` 状态

### 5.3 文本分块能力

任务：
- 实现 token 计数
- 实现基础 token 分块
- 实现 overlap 保留
- 预留语义合并逻辑

建议执行策略：
- P0 先完成固定 token 分块
- P1 再开启语义相似度合并

验收标准：
- 长文档可稳定拆成多个 chunk
- chunk 长度分布合理，无无限循环或空块

### 5.4 向量化与 ChromaDB

任务：
- 初始化 Chroma 持久化客户端
- 实现批量 embeddings
- 实现 chunk 写入
- 实现按 query 向量检索
- 实现按文档删除向量数据

补充要求：
- metadata 至少包含 `doc_id`、`chunk_index`、`filename`
- 对 embedding 请求失败增加异常处理

验收标准：
- 上传后 Chroma 中可查询到 chunk
- 删除文档后对应向量同步删除

### 5.5 BM25 检索

任务：
- 实现中文/英文简易分词
- 实现 `BM25Retriever`
- 提供获取全量 chunk 的接口

增强要求：
- 增加进程内缓存
- 文档新增或删除后刷新 BM25 索引

验收标准：
- 关键词查询可召回命中 chunk
- 检索耗时在小规模数据下可接受

### 5.6 混合检索与 Prompt 构建

任务：
- 实现 Query Rewriting
- 实现向量检索和 BM25 结果合并
- 实现去重和 TopK 截断
- 实现 RAG Prompt 构建

补充要求：
- Query Rewrite 失败时回退原始 query
- Prompt 中明确要求“无依据不回答”
- 检索结果保留来源信息以便后续展示引用

验收标准：
- 检索结果可用于回答问题
- 无知识命中时能明确说明而非编造

### 5.7 文档接口

任务：
- 实现 `POST /api/documents/upload`
- 实现 `GET /api/documents/`
- 实现 `DELETE /api/documents/{doc_id}`

补充要求：
- 统一返回格式
- 对不存在文档返回 404
- 删除时同时清理向量索引

验收标准：
- 前端可完成上传、列表、删除
- 状态与 chunk_count 展示正常

## 6. 第三阶段：聊天与记忆能力

### 6.1 LLM 客户端封装

任务：
- 实现 `get_openai_client()`
- 支持从配置读取 API Key、Base URL、模型名
- 兼容 OpenAI、DashScope、本地兼容接口

验收标准：
- 同一套客户端可被聊天、embedding、摘要、Code Review 复用

### 6.2 SSE 聊天接口

任务：
- 实现 `GET /api/chat/stream`
- 返回首包 `session_id`
- 返回流式内容 chunk
- 结束返回 `[DONE]`

补充要求：
- 设置正确 `text/event-stream` 响应头
- 正确处理中途失败和客户端断连
- 控制单 worker 运行

验收标准：
- 前端能实时显示 AI 逐字输出
- 首次请求可获得新 `session_id`

### 6.3 短期记忆

任务：
- 实现 `ShortTermMemory`
- 维护会话消息历史
- 超限时执行摘要压缩
- 组装发送给模型的上下文消息

补充修正：
- `summary_keep_rounds` 要真正生效
- 摘要只压缩较早轮次，不压最近对话

验收标准：
- 连续多轮问答上下文不断裂
- 长对话不会无限增长导致超限

### 6.4 长期记忆

任务：
- 实现结构化信息抽取
- 保存长期记忆到 SQLite
- 生成记忆 embedding
- 支持按 query 语义召回

补充要求：
- 先做文本精确去重，再做向量相似去重
- 增加 `custom_fact` 预留能力或扩展字段方案

验收标准：
- 用户表达明确偏好后，后续问答可被召回利用

### 6.5 长期记忆接口

任务：
- 实现 `GET /api/chat/memories/{user_id}`
- 实现 `DELETE /api/chat/memories/{memory_id}`

补充要求：
- 删除操作只删除指定记忆
- 返回列表时按创建时间倒序

验收标准：
- 前端能展示和删除长期记忆

### 6.6 会话持久化

任务：
- 会话首次创建时写入 `Conversation`
- 每轮 user/assistant 消息写入 `Message`
- 支持根据 `session_id` 重建上下文

说明：
- 这是对规范的工程化补齐
- 不做这一项则服务重启后会话丢失

验收标准：
- 服务重启后会话可恢复主要历史

### 6.7 异步记忆提取优化

任务：
- 将长期记忆提取改为后台任务
- 避免主聊天请求在收尾阶段阻塞

实现建议：
- 使用 `BackgroundTasks` 或安全的异步调度方式

验收标准：
- SSE 完成后接口能快速结束
- 记忆保存不阻塞主响应
